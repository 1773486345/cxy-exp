# 方向 A：模式感知的异常证据评分

记录日期：2026-06-30
更新日期：2026-07-13

方向定位：

```text
Pattern-Aware Anomaly Evidence Scoring
模式感知的异常证据评分
```

## 研究动机

真实多变量时序系统中的偏离并不具有固定含义。均值漂移、方差变化、周期强度改变、负载波动和变量间协同变化都会改变观测偏离的背景尺度，使得相同幅度的偏离在不同状态下具有不同风险含义。在稳定运行阶段，一个较小偏离可能已经具有异常指示性；而在剧烈波动阶段，相同偏离可能仍属于正常变化范围。

因此，多变量时序异常检测的关键并不只是度量偏离幅度，还在于理解偏离发生的结构背景。一个偏离可能出现在缓慢趋势变化中，也可能出现在短时突发扰动、局部高波动、周期结构改变或多变量同步变化中；这些背景模式会直接影响偏离是否应被视为异常。若将所有偏离统一压缩为单一分数并使用固定阈值判别，检测结果容易受到正常动态变化干扰，也难以准确保留真正具有异常意义的变化。

本方向要回答的问题是：如何对观测偏离进行模式感知的解释、校准与评分，使异常分数不仅反映偏离大小，也反映偏离所处的时序结构模式及其判别意义。

## 研究现状

多变量时序异常检测通常以观测序列与正常行为估计之间的差异作为异常依据，典型形式包括预测误差、重构误差、关联差异和表示距离。MSCRED 以多尺度 signature matrices 表征系统状态；GDN 通过图结构学习变量间依赖关系；Anomaly Transformer 将异常判别扩展到时间点关联模式差异；DCdetector 则通过多尺度双注意力和对比学习构造判别式异常表征。这些研究表明，有效异常证据不局限于单一时域残差，也可以来自多尺度结构、变量关系和关联模式。

另一方面，鲁棒分解、频域分析和非平稳建模说明，趋势、周期、频率、尺度和运行状态会改变观测偏离的含义。现有方法往往将这些信息隐式融合为统一判据，较少直接建模“同一幅度偏离在不同动态模式下为何具有不同异常语义”。因此，模式感知的条件异常评分仍是独立且未被充分解决的问题。

原始研究设定还考虑了与数值序列对齐的文本或其他外生模态：过程说明、运行状态、天气、控制命令、维护记录、变量语义和领域知识都可能提供仅由数值窗口难以恢复的正常机制约束。它们不是用来替代时序证据，也不能作为标签代理；其作用是条件化“此时此地何种数值关系、变化尺度和不确定性合理”。

## 问题定义

给定多变量窗口 `X`，模型应当判断某变量在时刻 `t` 的观测是否与正常条件机制兼容，而不只判断其重构误差是否大。若时刻 `t` 前存在真正可用的外生或语义证据 `M_{<=t}`，它可以作为辅助条件来消除数值窗口本身无法辨别的机制歧义，而不是替代异常检测任务：

```text
局部稳定性：       同样偏离在稳定与高波动阶段的含义不同。
趋势与突变：       缓慢演化和不受支持的突发变化不能混为一类。
多尺度结构：       短时扰动、持续偏移和尺度间不一致需要不同证据。
跨变量关系：       正常协同变化与关系失配的异常应当可区分。
```

结构背景不是异常类型标签，而是解释条件偏离所需的正常机制状态。

## 辅助条件证据的可扩展性

方向 A 的中心是多变量时序异常检测，不是多模态建模。数值时序的局部模式、跨变量动态关系和目标盲条件似然构成主模型与主要异常证据：

```text
p(x[t,d] | X_visible)
```

多模态只是可选手段。当额外证据能够说明数值窗口无法辨别的运行状态、允许的变量关系或变化尺度时，模型才扩展为：

```text
p(x[t,d] | X_visible, M_available(t))
```

这里的 `M_available(t)` 是评分时刻实际可获得的辅助条件子集。它不能成为独立的异常分类捷径；缺失时模型必须自然退化为数值核心，而不是用虚假的填充值制造确定性。

### 何时值得引入辅助模态

引入一个模态前，必须先回答它为异常检测解决什么数值歧义。合格的模态不是“信息更多”，而是能改变对正常机制的条件判断，例如：

```text
天气/负载/控制状态：
  判断当前波动或协同变化是否是外部状态允许的正常响应。

设备拓扑/变量语义/工艺规则：
  限制哪些变量关系、传播方向或组合在正常状态下合理。

维护、批次或模式切换记录：
  解释何时允许均值、方差或关系机制发生制度性变化。

新增传感观测：
  在数值指标仍含歧义时，提供对设备、环境或操作状态的独立观测。
```

若一个候选模态不能对应上述某种可检验的数值歧义，就不应为了“多模态”而接入。

### 模态角色，而非文件格式

不同模态进入模型的位置由其机制角色决定，而不是由它是否以文本 CSV 存储决定：

```text
静态机制先验：
  变量名称/单位、设备拓扑、工艺说明、系统说明、已知约束。
  作用：初始化变量 token、关系图先验或解码器的共享机制条件；
  不应被重复地伪装成逐时刻动态信号。

时间对齐的外生状态：
  天气、负载、控制命令、运行模式、可观测环境量。
  作用：以 past-only 状态 token 调制局部模式表征、动态图的边参数
  和条件尺度，使模型区分“数值偏离”与“外部状态允许的变化”。

异步事件与记录：
  维护工单、告警、人工日志、批次切换、模式变更。
  作用：按真实产生时间和可见延迟形成事件 token；它可以改变允许的
  转移或不确定性，但不能使用事后归因、故障结论或标签性描述。

新增传感模态：
  图像、音频、波形、日志计数或其他设备侧观测。
  作用：先编码为带时间戳和可用性掩码的状态 token，再作为上述条件的一类；
  不为“多模态”而新增一个独立末端分数。
```

因此，频域、趋势和多尺度统计仍是从 `X` 派生的数值模式，不应被包装成独立外生模态；真正的外生/语义模态只在能够改变模型对正常机制的条件判断时才有价值。

### 目标模型形态

后续 `PatternAD` 应保持一个完整、可独立运行的目标盲数值机制核心；机制条件器是按需叠加的辅助路径：

```text
数值窗口 X_visible
    -> 局部模式编码 + 多尺度动态关系图
静态/动态/事件模态 M_available(t)
    -> 按角色编码 + 可用性/时间戳掩码（可选）
    -> 仅在可用时调制变量表征、关系先验/边状态和条件不确定性
    -> 目标盲条件解码
    -> 数值核心或条件扩展的异常分数
```

文本只是当前已有的一种候选载体，不是模型身份。对文本应采用离线、可版本化的编码或小型适配器：静态描述按实体或变量编码一次；逐时刻文本仅在来源和时间对齐可靠、并能解释特定数值歧义时编码为状态 token。不能恢复逐窗口调用大模型/tokenizer 的旧路径，也不能把重复的通用 prompt 误当作新的观测信息。

### 可用性、来源与评估

以下是辅助模态进入异常检测模型的最小科学约束：

1. 记录每个模态的来源、时间戳、可见延迟、粒度、缺失率及其与标签的关系；评分只读取时刻 `t` 前实际可用的部分。
2. 任何含有事后故障结论、攻击名称、异常标签或由目标窗口生成的解释的内容都不能作为同一时刻的条件输入。它可以保留为数据元信息或用于审计。
3. 模态必须真实影响条件均值、关系状态或条件尺度；只在 benchmark 接口中传递、或在最后拼接一个相似度/分数，不构成对异常检测机制的有效帮助。
4. 每个候选模态先与数值核心比较其所针对的歧义是否改善；消融按机制角色与可用性进行，而不是只做一次笼统的 `text_off/text_on`。同时报告模态缺失、随机化或延迟后的退化，防止把相关但不可部署的信息误当作贡献。

当前的文本 CSV 和恢复的 `PatternAD*.sh` 都保留 `--text-name-list`，它是现有 benchmark 的文本载体，而不是方向 A 的模态上限。当前数据中混有静态说明、天气描述、重复通用 prompt 和 placeholder；在建立来源、可用性和“它解决何种数值歧义”的清单前，不能把它们统一当作有效外生证据，也不能声称多模态增益。

## 当前模型

当前唯一活跃模型为 `PatternAD`，代码位于：

```text
PatternAD-main/ts_benchmark/baselines/PatternAD/
```

当前实现是目标盲的多尺度动态关系条件检测器：共享时序编码器为每个变量和尺度形成表征，并在每个时间点推断有向跨变量关系状态。对被掩码的目标值，时间分支和关系分支共同给出条件高斯分布；最终分数为条件负对数似然，而不是手工拼接的残差、尾部统计或局部统计加权。

现有 `PatternAD` 已实现 `detect_fit` / `detect_score` 与 benchmark 所需的 `detect_multi_fit` / `detect_multi_score`，因此保留 `--text-name-list` 的既有脚本可以运行。当前文本输入做严格对齐校验和诊断记录，但尚未作为条件特征参与评分；这避免将未审计 prompt 或 placeholder 误写成多模态贡献。未来的结构化外生状态、事件记录或其他传感模态只有在能服务上述异常检测动机时才进入模型。

```text
可见数值窗口与目标掩码
    -> 共享多尺度时序表征
    -> 多尺度动态有向关系图
    -> 目标盲条件解码
    -> 联合条件似然异常分数
```

模型的三个预声明结构消融为：

```text
full          完整的多尺度动态关系图。
single_scale  移除多尺度关系融合。
no_graph      移除全部跨变量消息，保留相同的时间骨干与概率头。
```

目标值在训练和推理时都被掩码，关系分支不能读取被解释的终点值。这样，跨变量机制不是评分后的修正项，而是表征、解码和训练目标的一部分。

### 实现状态（2026-07-13）

- 当前代码是与上述动机一致的待验证候选模型，而不是已经被实验结果证明优于原始 `PatternAD` 或既有基线的结论。多尺度条件关系、目标盲解码和条件似然构成需要通过完整对比与消融验证的设计贡献。
- 已通过 `tests.test_patternad_core` 与 `tests.test_anomaly_protocol` 共 18 项 CPU 契约测试；所有既有 `PatternAD*.sh` 已通过 Shell 语法检查。尚未启动真实数据 GPU 实验。
- 脚本维持原有目录和单 GPU 前台顺序执行方式。按单任务约 30 GB 显存预算设置默认训练/评分批量：Daphnet、Energy、GECCO、MSDS、SKAB、Weather 为 `128/256`；Genesis、MetroPT3 为 `96/192`；SMD 为 `64/128`；HAI21 为 `48/96`。后一个数值是仅影响推理吞吐的 `score_conditioning_batch_size`。

## 评估原则

1. 训练、模型选择和阈值不能使用测试标签；所有模态均受时间可用性和反泄漏约束。
2. 先在固定开发集运行 `full`、`single_scale`、`no_graph`，再冻结结构和超参数。
3. 冻结后与重构、图、Transformer 和概率 TSAD 基线在相同数据划分下比较。
4. 报告 AUC-PR、AUC-ROC、VUS-PR、VUS-ROC、运行时间、参数量，以及多 seed 均值和标准差。
5. 真实数据的 GPU 实验尚未运行；目前已完成数值核心、目标盲和 benchmark multi-input 接口的 18 项 CPU 契约测试。数值核心可以进行正式真实数据实验；每个候选辅助模态完成来源/可用性/数值歧义审计和对应消融协议后，才进入其正式对比实验。
