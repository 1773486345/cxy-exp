# 方向 A：面向非平稳时序异常检测的异常保持分解

记录日期：2026-06-30
更新日期：2026-07-17

## 方向定位

```text
Anomaly-Preserving Decomposition for Nonstationary Time-Series Anomaly Detection
面向非平稳时序异常检测的异常保持分解
```

## 研究动机

真实多变量时序系统中的偏离并不具有固定含义。均值漂移、方差变化、周期强度改变、负载波动和变量间协同变化都会改变观测偏离的背景尺度，使得相同幅度的偏离在不同状态下具有不同风险含义。在稳定运行阶段，一个较小偏离可能已经具有异常指示性；而在剧烈波动阶段，相同偏离可能仍属于正常变化范围。

因此，多变量时序异常检测的关键并不只是度量偏离幅度，还在于理解偏离发生的结构背景。一个偏离可能出现在缓慢趋势变化中，也可能出现在短时突发扰动、局部高波动、周期结构改变或多变量同步变化中；这些背景模式会直接影响偏离是否应被视为异常。若将所有偏离统一压缩为单一分数并使用固定阈值判别，检测结果容易受到正常动态变化干扰，也难以准确保留真正具有异常意义的变化。

本方向要回答的问题是：如何对观测偏离进行模式感知的解释、校准与评分，使异常分数不仅反映偏离大小，也反映偏离所处的时序结构模式及其判别意义。

研究难点不是简单地把预测领域已有的趋势、周期或频率分解迁移到异常检测。预测只要求分解后的分量重组后降低平均预测误差；异常检测还要求分解能够抑制合法的非平稳变化，同时不能把异常吸收到趋势、频带、归一化统计或状态路由中。换言之，分解必须具有“异常保持”性质。

## 研究现状

多变量时序异常检测通常以观测序列与正常行为估计之间的差异作为异常依据，典型形式包括预测误差、重构误差、关联差异和表示距离。MSCRED 以多尺度 signature matrices 表征系统状态；GDN 通过图结构学习变量间依赖关系；Anomaly Transformer 将异常判别扩展到时间点关联模式差异；DCdetector 则通过多尺度双注意力和对比学习构造判别式异常表征。这些研究表明，有效异常证据不局限于单一时域残差，也可以来自多尺度结构、变量关系和关联模式。

另一方面，鲁棒分解、频域分析和非平稳建模说明，趋势、周期、频率、尺度和运行状态会改变观测偏离的含义。现有方法往往将这些信息隐式融合为统一判据，较少直接建模“同一幅度偏离在不同动态模式下为何具有不同异常语义”。因此，模式感知的条件异常评分仍是独立且未被充分解决的问题。

原始研究设定还考虑了与数值序列对齐的文本或其他外生模态：过程说明、运行状态、天气、控制命令、维护记录、变量语义和领域知识都可能提供仅由数值窗口难以恢复的正常机制约束。它们不是用来替代时序证据，也不能作为标签代理；其作用是条件化“此时此地何种数值关系、变化尺度和不确定性合理”。

2024 至 2026 年的工作进一步说明，时频表示、自适应分解和测试时适应正在成为非平稳 TSAD 的重要路线，但现有方法仍没有直接解决本方向的核心约束：分解器、归一化器和路由器是否在判断 `x[t,d]` 时读到了 `x[t,d]` 本身。只要答案是“是”，异常就可能改变用于解释自己的正常机制。

## 核心问题

对变量 `d` 在时刻 `t` 的观测进行评分时，先由目标盲可见上下文 `C[t,d]` 推断当前正常机制：

```text
C[t,d] = {x[τ,d] | τ < t} ∪ {评分时确实可见的其他变量/外生条件}

D(C[t,d]) -> 当前正常机制的受限视角或状态
p(x[t,d] | D(C[t,d])) -> 一个条件正常分布
```

最终问题是：

```text
x[t,d] 是否与可见历史支持的当前正常机制兼容？
```

而不是：

```text
能否从同一个残差构造多个分数，再把它们加起来？
```

频率、趋势、原始数值 patch 和多尺度摘要是同一数值序列的不同“视角”，不是统计独立的模态。它们不能被默认当作相互独立的异常证据。最终模型应使用这些视角共同生成一个条件正常解释，而不是分别产生分数后进行无约束融合。

## 可见性与异常保持约束

同步评分 `x[t,d]` 时，任何参与预测其正常值的分支都不得读取该目标。目标变量的归一化、趋势、频谱、分解边界和状态路由只能由过去数据计算；若关系分支使用同时刻其他变量，则只能看到 `x[t,-d]`。

```text
目标变量可见历史：       x[<=t-1, d]
可选同步关系证据：       x[t, -d]
待评分目标：             x[t, d]，对所有解释分支隐藏
```

因此需要满足以下约束：

1. 所有均值、尺度和 RevIN 类统计只由目标历史计算，不能包含目标点。
2. 频谱与分频掩码只由目标历史计算，不能对包含目标点的整窗做 FFT 后再评分。
3. 多尺度或专家路由只由目标历史状态决定，不能让异常选择最容易重构自己的路径。
4. 在线状态更新必须筛除疑似异常；不能用整个测试流无条件更新趋势、原型或模型。
5. 所有视角最终形成一个条件分布或一个联合校准分数，不能假设分数独立。

因果周期视角对新周期或渐变异常天然存在检测延迟，因为它需要一段历史确认结构变化。该延迟应当作为结果报告，而不是通过读取未来数据消除。

## 外生信息与“多模态”的边界

方向 A 的核心仍是多变量数值时序异常检测，不依赖外生信息才能成立。文本、天气、控制状态、维护事件、拓扑和变量语义只有在能够消除纯数值历史无法消除的正常机制歧义时才进入模型：

```text
p(x[t,d] | X_visible)

必要且真实可用时扩展为：

p(x[t,d] | X_visible, M_available(t))
```

其中 `M_available(t)` 必须在评分时真实可见，并记录来源、时间戳、可见延迟、粒度和缺失率。由目标窗口自动生成的文本不是新的观测模态；包含事后故障结论、攻击名称或标签性描述的内容不能作为输入。

静态变量名、单位、拓扑和工艺描述可以作为机制先验；时间对齐的负载、天气、控制命令可以作为外生状态；维护记录只能按真实产生时间形成事件条件。它们应调制条件均值、允许的关系或条件尺度，不能在末端另加一个“文本异常分数”。当前文本 CSV 混有静态说明、重复 prompt 和 placeholder，因此继续保留接口与审计，但不参与当前数值核心。

## 近期工作与可复用内容（2024-2026）

以下只记录对方向 A 有直接作用的原始论文和官方实现，不把“近期”本身当作采用理由。

### ADAD：自适应分解是最接近的直接启发

[ADAD（IEEE Transactions on Big Data, 2026）](https://doi.org/10.1109/TBDATA.2026.3695421)指出，固定移动平均忽略不同变量和不同时刻的演化模式。其核心是把频域分解写成按变量自适应的频谱切分，并通过跨分量连接让一个分量指导另一个分量的模式学习。

可借用：

- 按变量、按状态学习分频边界，而不是统一移动平均或固定频带。
- 不把分量完全独立处理，而是显式约束跨分量的条件作用。

不能直接照搬：

- 本方向必须把分频边界改为由 `x[<=t-1,d]` 决定，并验证改变 `x[t,d]` 不会改变分解。
- 论文的自适应分解本身不等于异常保持；目标可见性仍需单独审计。

### CATCH：频率分块与按频带通道选择可拆出来复用

[CATCH（ICLR 2025）](https://arxiv.org/abs/2410.12261)把 FFT 的实部和虚部按频率轴分 patch，并使用带 Gumbel-Softmax 的通道掩码生成器，为每个频率 patch 选择相关变量；其[官方实现](https://github.com/decisionintelligence/CATCH)已经包含频率分块、通道掩码和频域重构损失。

可借用：

- `rFFT/FFT -> 频率 patch -> 频带级编码` 的数据组织方式。
- 每个频带拥有不同跨变量相关性的思想。
- 滑动局部频谱误差映射回点级时间轴的计算实现。

不能直接照搬：

- 官方实现对完整窗口做 RevIN、FFT 和重构，并使用 `时域 MSE + 固定权重 × 频域误差` 评分，仍可能读取并重构待评分异常。
- 双层优化和通道图不是第一版必需模块；此前动态图负结果要求先证明分解本身有效。

### TFMAE 与 Dual-TF：借训练约束和粒度对齐，不借末端分数拼接

[TFMAE（ICDE 2024）](https://doi.org/10.1109/ICDE60146.2024.00099)分别进行时间窗口掩码和基于幅值的频率掩码，并使用时频表示的对比约束，目的在于降低异常污染与分布偏移对重构学习的影响；[官方代码](https://github.com/LMissher/TFMAE)可用于参考掩码和对比目标。

[Dual-TF（The Web Conference 2024）](https://doi.org/10.1145/3589334.3645556)指出频域窗口分数与时域点分数存在粒度不一致，使用外层/内层嵌套滑窗将两者对齐到更精细的异常边界。

可借用：

- 仅在正常训练数据上做“遮蔽某一时间片或频带并预测”的自监督约束。
- 任何窗口级频率量都必须定义清楚怎样映射回 `t`；最简单的规避方式是直接做 past-to-next-point 条件预测。

不能直接照搬：

- 时域与频域表示接近不等于它们对异常具有互补性。
- 不采用固定 `time_score + λ × frequency_score` 作为方向 A 的核心贡献。

### TimeMixer、Pathformer 与 TimeMixer++：只借动态尺度选择思想

[TimeMixer（ICLR 2024）](https://openreview.net/forum?id=7oLshfEIC2)在不同下采样尺度分别分解季节和趋势，并进行细到粗、粗到细的跨尺度混合。[Pathformer（ICLR 2024）](https://openreview.net/forum?id=lJkOCMP2aW)使用多种 patch 尺度专家和稀疏路由，根据输入动态选择路径。[TimeMixer++](https://arxiv.org/abs/2410.16032)进一步组合多分辨率时频成像、趋势/季节分解和自适应多分辨率混合。

可借用：不同状态应选择不同时间尺度，而不是所有样本固定相加所有尺度；稀疏路由及其负载均衡损失可以作为后续状态路由实现参考。

不能直接照搬：这些机制主要为预测或通用分析设计，路由通常读取完整输入窗；在 TSAD 中，路由必须由目标盲历史生成，否则异常会选择更容易解释自己的尺度专家。

### M2N2 与 CANDI：非平稳适应必须防止异常污染

[M2N2（AAAI 2024）](https://doi.org/10.1609/aaai.v38i12.29210)用指数移动均值估计测试流趋势，并仅以模型当前判断为正常的样本进行在线更新。这说明“新正常”问题应被显式处理，但其趋势统计仍由整个流更新，异常污染风险没有完全消失。

[CANDI（AAAI 2026）](https://doi.org/10.1609/aaai.v40i17.38524)进一步从正常验证集保存高分误报的潜表示；测试高分样本只有在潜空间接近这些已知误报时才参与适应，并只更新轻量 SANA 残差适配器，冻结原检测器。其误报筛选、Mahalanobis 距离和门控残差适配器可作为未来适应阶段的参考。

当前不做测试时适应：在静态目标盲分解尚未成立前，引入适应无法区分收益来自分解还是在线更新。

## 主干与基座选择

### 选择：CATCH 频率分块与通道融合主干

本方向不再因实现简单而把 PatchTST 设为默认主干。当前可公开复现、直接面向多变量 TSAD、并且与频率分解动机最契合的强主干是 [CATCH（ICLR 2025）](https://arxiv.org/abs/2410.12261)。其论文在 12 个真实多变量数据集和 6 类合成异常上比较 15 个基线；完整 AUC-PR、R-AUC-PR、VUS-PR 表中，CATCH 获得 31 个第一名计数。它不是每个数据集都最优，例如 SWaT 上仍有明显短板，但现有公开证据比 PatchTST、ModernTCN 和通用预测主干更适合成为唯一工程基座。

选择 CATCH 的理由不是“频域模块更多”，而是其主干已经把两个与方向 A 直接相关的能力结合起来：

```text
频率轴分 patch
    -> 在不同频率 patch 上分别学习跨变量相关性
    -> 隔离不相关变量，聚合相关变量
    -> 重建时频结构并形成异常证据
```

其官方实现公开了频率 patch、Gumbel-Softmax 通道掩码、masked attention、频域辅助损失和训练流程，可以作为工程参照。复杂度不作为排除它的理由。

### 为什么仍需大改，而不是直接运行 CATCH

CATCH 官方异常检测路径对包含待评分点的完整窗口执行 RevIN、FFT 和全窗重构，并将时域 MSE 与固定权重频域误差相加。这与方向 A 的异常保持约束不一致：异常可以影响归一化统计、频谱、通道掩码和自身重构。

因此保留的是 CATCH 的高性能表示主干，不保留其目标可见的输入/评分协议：

```text
保留：频率 patch、频带级通道融合、masked attention、强频域编码器
替换：全窗重构 -> past-to-next-point 条件预测
替换：全窗 RevIN -> past-only 归一化
替换：固定时频分数相加 -> 一个条件分布分数
```

### 其他主干的定位

[TimeMixer++（ICLR 2025）](https://arxiv.org/abs/2410.16032)的多分辨率混合仅在 CATCH 主干明确显示频率尺度不足时借用，当前不并行开发。[PatchTST](https://github.com/yuqinie98/PatchTST)和 [ModernTCN](https://openreview.net/forum?id=vpJMJerXHU)保留为正式阶段的强主干基线；[MOMENT](https://proceedings.mlr.press/v235/goswami24a.html)只作为后期冻结预训练表示对照。ADAD 是自适应分解机制来源，但没有公开官方实现，不适合作为当前工程基座。

## 历史候选：APD-CATCH v1（已停止）

v1 是以 CATCH 为工程主干的第一版目标盲条件预测模型，而非原版 CATCH 的直接复现。

### 1. Past-only 归一化

对每个变量只用历史窗口计算位置和尺度：

```text
m[t,d] = mean(x[t-L:t-1,d])
s[t,d] = std(x[t-L:t-1,d]) + epsilon
u[t-L:t-1,d] = (x - m[t,d]) / s[t,d]
```

首版不使用包含 `x[t,d]` 的 RevIN，也不在线更新全局统计。

### 2. 由历史状态产生可微分频边界

历史编码 `h[t,d]` 输出该变量当前状态下的归一化截止频率 `c[t,d]`：

```text
F[t,d] = rFFT(u[t-L:t-1,d])
c[t,d] = sigmoid(g(h[t,d]))

M_low(f) = sigmoid((c[t,d] - f_norm) / tau)
M_high(f) = 1 - M_low(f)

V_low  = irFFT(M_low  * F)
V_high = irFFT(M_high * F)
```

双频带而不是四视角，是为了只检验一个问题：按变量和状态自适应的分频是否比固定分频更能区分合法非平稳变化与异常。`M_low + M_high = 1` 保证信息完备；掩码平滑度和截止频率边界约束防止数值退化。

### 3. 共享 CATCH 频率通道主干与一个条件分布

低频和高频分量使用同一个 CATCH-style 编码器。每个分量先沿频率轴分 patch，再在每个 patch 内生成通道掩码并执行 masked channel attention，最后聚合所有频率 patch：

```text
h_low  = CATCHEncoder(FrequencyPatch(V_low), past_only=True)
h_high = CATCHEncoder(FrequencyPatch(V_high), past_only=True)

mu_low, mu_high = component heads(h_low, h_high)
mu[t,d] = m[t,d] + s[t,d] * (mu_low + mu_high)
sigma[t,d] = positive_scale_head(h_low, h_high, history_state)
```

最终输出一个高斯条件负对数似然：

```text
score[t,d] = 0.5 * ((x[t,d] - mu[t,d]) / sigma[t,d])^2 + log sigma[t,d]
```

通道融合是 CATCH 主干的固有组成，不再另建 PatternAD 动态图；它的掩码只由过去频率 patch 生成。首版不加入 normal-mode memory、文本、测试时适应、频域独立分数或最大值融合。

### 4. 训练约束

核心训练目标是正常训练段上的一步条件似然。保留 CATCH 的通道掩码离散化和相关/无关通道分离约束；分解侧只增加频带掩码平滑约束和截止频率边界约束。必须验证：固定 `x[<=t-1]` 后任意改变 `x[t,d]`，得到的 `m`、`s`、`c`、`M_low`、`M_high`、`mu` 和 `sigma` 完全不变。

### 5. 最小实验门控与结果

受控机制实验固定数据、参数预算、训练轮数和评分协议，比较：

```text
causal_catch  目标盲 CATCH 主干，不做低/高频分解
fixed          相同 CATCH 主干与双频带结构，使用固定截止频率
adaptive       相同 CATCH 主干，由 past-only 历史状态生成截止频率
```

门槛为：正常 FPR 不超上限；`adaptive` 的混合异常平均 AP 高于 `causal_catch`；高于 `fixed`；目标盲不变量通过。三个随机种子、名义 1% FPR 的结果为：

| 版本 | 正常 FPR | Spike AP | Level AP | Periodic AP | Relation AP | 四类平均 AP |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| `causal_catch` | 0.0098 | 0.9204 | **0.1557** | **0.8099** | 0.1800 | 0.5165 |
| `fixed` | 0.0105 | 0.9376 | 0.1463 | 0.7664 | **0.1998** | 0.5125 |
| `adaptive` | **0.0086** | **0.9652** | 0.1356 | 0.7924 | 0.1833 | **0.5191** |

adaptive 虽以 0.0026 的绝对平均 AP 增益通过弱门槛，但 level 与 periodic 更差；不同随机种子的 cutoff 也不稳定。随后真实数据与 v1.1 尺度诊断均没有建立独立收益，因此停止对 cutoff、温度或尺度下界的继续调参。该阶段保留为 v2 的明确反例：可学习分频若没有不可替代职责，不能构成方向 A 的贡献。

## 模型主干与当前思路

工程基座选择 CATCH，而非另起预测模型：它已经实现频率轴分 patch、按频带学习通道关系以及
masked cross-channel Transformer。需要替换的是目标可见的训练/评分协议：

```text
保留：频率 patch、通道掩码、跨通道 Transformer、通道关系损失
替换：全窗重构 -> past-to-next-point 条件预测
替换：全窗 RevIN -> 训练期固定的稳健参考归一化
替换：时域分数 + 频域分数 -> 一个 Gaussian NLL
```

最终逐变量分数为：

```text
score[t,d] = 0.5 * ((x[t,d] - mu[t,d]) / sigma[t,d])^2 + log sigma[t,d]
```

测试集前 `seq_len` 个点历史不足，不参与指标计算。

### 当前候选：Causal-State-CATCH v2

v2 将可解释的慢状态和创新职责分开。先在无标签训练子段计算逐变量参考位置/尺度：MAD 为零
时回退训练标准差，仍为零时使用单位尺度；这些统计量在训练和测试流中固定。它吸收了 v1.1
“不能让局部近常数窗口把尺度降至 epsilon”的有效原则，但不把尺度下界当作冷启动变量的
正常性证据。

对历史窗口，非参数因果 EMA 以 `round(seq_len / 6)` 产生下一点慢状态，CATCH 只编码去慢
状态后的创新。三个同参数预算对照为：

```text
causal_catch: CATCH(standardized history) -> mu, sigma
state:        causal EMA -> state mean; CATCH(innovation) -> mean correction, sigma
state_scale:  state 的 mu;
              sigma = sqrt(learned innovation scale^2 + recent innovation scale^2)
```

`state_scale` 中近期创新只能拓宽条件尺度，不能缩小基础尺度。训练期恒定变量在测试期的首次
变化是否正常仍是上下文不足问题；未来可单独研究评分时可见控制状态或同步同伴变量，但必须
与纯历史结果分开报告。

## 模型迭代与已得结论

### 1. 后验残差组合：停止

早期方案从同一重构残差派生尺度、趋势、频率和同步分量，再做末端组合。它重复计数且没有
定义当前状态下的正常值和允许不确定性；原始残差控制总体更好。结论是状态应进入条件均值或
条件尺度的生成，不是残差后的加权。

### 2. PatternAD 目标盲动态图：停止

在修正为目标级 leave-one-variable-out 协议后，PSM 和 BATADAL dataset04 上的 `full` 与
`single_scale` 在 AUC-PR、VUS-PR 均低于同容量 `no_graph`。关系证据可以作为未来受控条件，
但当前动态图不是方向 A 的模型身份。

### 3. 固定多视角与最大值融合：停止

三种子受控机制实验覆盖合法均值/尺度/频率变化与 spike、level、periodic、relation 异常。
正常 FPR 在 1.1%--1.4%，但慢状态与周期原型没有形成预期互补性；局部视角平均 AP 为 0.777，
最大值融合降至 0.654。否定的是固定手工视角融合，不是否定异常保持分解问题。

### 4. APD-CATCH v1 自适应频率 cutoff：停止

v1 以目标盲历史产生低/高频软切分，对比 `causal_catch`、`fixed`、`adaptive`。合成门控中
adaptive 四类平均 AP 为 0.5191，causal_catch 为 0.5165，绝对增益仅 0.0026，且 level 与
periodic 更差。真实数据没有建立独立增益。原因是低/高频两路仍可由自由 head 互相替代，
没有不可替代职责。

### 5. v1.1 局部 MAD 尺度下界：保留为负结果

Genesis 的训练期多数变量窗口 MAD 与全局 MAD 都为零，尺度下界退化为 epsilon；测试新工况
仍产生近零预测尺度和极端正常 NLL。表面 AUC-ROC/AUC-PR 虽从约 0.40/0.0035 升至
0.710/0.226，三个变体却几乎同分，不能当作有效收益。停止 v1.1 的 PSM、SWAT 和多种子
扩展，也不继续调 epsilon、分位数或尺度上限。

### 6. v2：当前待验证

v2 保留 CATCH 主干，不再增加频率分支或并行模型。`causal_catch` 是每轮必跑的目标盲锚点；
任何新增机制只能以相同训练协议和参数预算与它比较。v2 已实现并通过核心不变量、配置和
运行器测试，但尚无真实数据结果。

## 实验计划与成本边界

### 首轮筛选

首轮固定为 CalIt2、GECCO、Genesis、NYC、PSM，三个变体和 `seed=2021`，共 15 个任务。
PSM 的旧 v1 每变体约需 2 小时训练与 17--22 分钟推理，成本可接受。

数据维度不是运行成本的充分代理。SMAP 虽与 PSM 同为 25 个变量，但其官方配置为 10 epoch、
3 层，测试段约为 PSM 的五倍；Creditcard、CICIDS、MSL、SMD、SMAP 应先仅运行
`causal_catch` 记录实际 wall time 和显存，再决定是否展开三个变体。SWAT 使用
`seq_len=2048`、`patch_size=256`、`batch_size=32`，当前明确不进入筛选。

### 稳定性与正式扩展

首轮按数据集独立判断：在同一数据集、同名指标上，任一 `state` 或 `state_scale` 相对原 CATCH
持平或提升，或者超过已有 baseline，即保留该数据集和变体作为有效结果。无需先要求所有数据集
方向一致，也不以多种子为首轮门槛；对表现有价值的数据集再按需要追加 `seed=2022/2023`。
通过成本测量的数据集可加入三变体扩展；ASD 的 12 个子序列在模型选择后作为独立数据族报告均值。
原 CATCH 与 v2 的可见性和阈值协议差异在结果中注明即可，只有需要声明严格的协议改进时才重跑。

## 评估原则

1. 每个数据集优先比较 AUC-ROC、AUC-PR 和 Aff-F；其余 R-AUC、VUS、运行时间和参数量同时保存即可。
2. 同一数据集上，v2 任一变体相对原 CATCH 持平/提升，或超过已有 baseline，即为该数据集的正向结果。
3. 连续排名指标用于比较，1% FPR 校准标签作为辅助工作点；多种子仅在需要时补充，不是首轮门槛。
4. 所有归一化、状态、分解和路由仍需通过目标可见性审计，避免把数据泄漏误当作性能提升。

范围内。它们会引入独立的因果或机制主张，应在 v2 的状态--创新分解成立后再讨论。
