# 方向 A：模式感知的异常证据评分

记录日期：2026-06-30
更新日期：2026-07-13

方向定位：

```text
Pattern-Aware Anomaly Evidence Scoring
模式感知的异常证据评分
```

## 研究动机

真实多变量时序系统中的偏离并不具有固定含义。均值漂移、方差变化、周期强度改变、负载波动和变量间协同变化都会改变观测偏离的背景尺度，使得相同幅度的偏离在不同状态下具有不同风险含义。在稳定运行阶段，一个较小偏离可能已经具有异常指示性；而在剧烈波动阶段，相同偏离可能仍属于正常变化范围。

因此，多变量时序异常检测的关键并不只是度量偏离幅度，还在于理解偏离发生的结构背景。一个偏离可能出现在缓慢趋势变化中，也可能出现在短时突发扰动、局部高波动、周期结构改变或多变量同步变化中；这些背景模式会直接影响偏离是否应被视为异常。若将所有偏离统一压缩为单一分数并使用固定阈值判别，检测结果容易受到正常动态变化干扰，也难以准确保留真正具有异常意义的变化。

本方向要回答的问题是：如何对观测偏离进行模式感知的解释、校准与评分，使异常分数不仅反映偏离大小，也反映偏离所处的时序结构模式及其判别意义。

## 问题定义

给定多变量窗口 `X`，模型应当判断某变量在时刻 `t` 的观测是否与正常条件机制兼容，而不只判断其重构误差是否大：

```text
局部稳定性：       同样偏离在稳定与高波动阶段的含义不同。
趋势与突变：       缓慢演化和不受支持的突发变化不能混为一类。
多尺度结构：       短时扰动、持续偏移和尺度间不一致需要不同证据。
跨变量关系：       正常协同变化与关系失配的异常应当可区分。
```

结构背景不是异常类型标签，而是解释条件偏离所需的正常机制状态。

## 当前模型

当前唯一活跃模型为 `MechanismGraphAD`，代码位于：

```text
PatternAD-main/ts_benchmark/baselines/MechanismGraphAD/
```

模型以共享时序 Transformer 为表征骨干，在每个时间点和时间尺度上推断变量间的动态有向关系状态。对被掩码的目标值，时间分支和关系分支共同给出条件高斯分布；最终分数为条件负对数似然，而不是手工拼接的残差或尾部统计。

```text
共享时序表征
    -> 多尺度动态有向关系图
    -> 目标盲条件解码
    -> 联合条件似然异常分数
```

模型的三个预声明结构消融为：

```text
full          完整的多尺度动态关系图。
single_scale  移除多尺度关系融合。
no_graph      移除全部跨变量消息，保留相同的时间骨干与概率头。
```

目标值在训练和推理时都被掩码，关系分支不能读取被解释的终点值。这样，跨变量机制不是评分后的修正项，而是表征、解码和训练目标的一部分。

## 评估原则

1. 训练、模型选择和阈值不能使用测试标签。
2. 先在固定开发集运行 `full`、`single_scale`、`no_graph`，再冻结结构和超参数。
3. 冻结后与重构、图、Transformer 和概率 TSAD 基线在相同数据划分下比较。
4. 报告 AUC-PR、AUC-ROC、VUS-PR、VUS-ROC、运行时间、参数量，以及多 seed 均值和标准差。
5. 真实数据的 GPU 实验尚未运行；目前仅完成 CPU 接口与目标盲契约测试。

运行入口：

```bash
cd PatternAD-main
bash scripts/mechanism_graph_ad/run_real_benchmark_v1.sh GPU_ID
```
