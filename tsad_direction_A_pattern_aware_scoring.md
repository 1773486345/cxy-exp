# 方向 A：模式感知的异常证据评分

记录日期：2026-06-30
更新日期：2026-07-13

方向定位：

```text
Pattern-Aware Anomaly Evidence Scoring
模式感知的异常证据评分
```

## 研究动机

真实多变量时序系统中的偏离并不具有固定含义。均值漂移、方差变化、周期强度改变、负载波动和变量间协同变化都会改变观测偏离的背景尺度，使得相同幅度的偏离在不同状态下具有不同风险含义。在稳定运行阶段，一个较小偏离可能已经具有异常指示性；而在剧烈波动阶段，相同偏离可能仍属于正常变化范围。

因此，多变量时序异常检测的关键并不只是度量偏离幅度，还在于理解偏离发生的结构背景。一个偏离可能出现在缓慢趋势变化中，也可能出现在短时突发扰动、局部高波动、周期结构改变或多变量同步变化中；这些背景模式会直接影响偏离是否应被视为异常。若将所有偏离统一压缩为单一分数并使用固定阈值判别，检测结果容易受到正常动态变化干扰，也难以准确保留真正具有异常意义的变化。

本方向要回答的问题是：如何对观测偏离进行模式感知的解释、校准与评分，使异常分数不仅反映偏离大小，也反映偏离所处的时序结构模式及其判别意义。

## 研究现状

多变量时序异常检测通常以观测序列与正常行为估计之间的差异作为异常依据，典型形式包括预测误差、重构误差、关联差异和表示距离。MSCRED 以多尺度 signature matrices 表征系统状态；GDN 通过图结构学习变量间依赖关系；Anomaly Transformer 将异常判别扩展到时间点关联模式差异；DCdetector 则通过多尺度双注意力和对比学习构造判别式异常表征。这些研究表明，有效异常证据不局限于单一时域残差，也可以来自多尺度结构、变量关系和关联模式。

另一方面，鲁棒分解、频域分析和非平稳建模说明，趋势、周期、频率、尺度和运行状态会改变观测偏离的含义。现有方法往往将这些信息隐式融合为统一判据，较少直接建模“同一幅度偏离在不同动态模式下为何具有不同异常语义”。因此，模式感知的条件异常评分仍是独立且未被充分解决的问题。

原始研究设定还考虑了与数值序列对齐的文本或其他外生模态：过程说明、运行状态、天气、维护记录或领域知识可以提供仅由数值窗口难以恢复的正常机制约束。它们不是用来替代时序证据，也不能作为标签代理；其作用是条件化“此时此地何种数值关系与变化模式合理”。

## 问题定义

给定多变量窗口 `X`，以及在时刻 `t` 前可获得的文本或外生上下文 `Z`，模型应当判断某变量在时刻 `t` 的观测是否与正常条件机制兼容，而不只判断其重构误差是否大：

```text
局部稳定性：       同样偏离在稳定与高波动阶段的含义不同。
趋势与突变：       缓慢演化和不受支持的突发变化不能混为一类。
多尺度结构：       短时扰动、持续偏移和尺度间不一致需要不同证据。
跨变量关系：       正常协同变化与关系失配的异常应当可区分。
```

结构背景不是异常类型标签，而是解释条件偏离所需的正常机制状态。

## 多模态约束

`--text-name-list` 是既有 benchmark 的正式输入，不是可以因当前时序主干尚未消费文本而删除的兼容参数。静态文本可描述设备、变量语义和正常过程约束；逐时刻文本可描述事前可用的天气、工况或维护上下文。二者都必须与数值序列的训练、校准和测试切分一致。

多模态分支必须遵守以下边界：

1. 文本只能使用时刻 `t` 前可获得的信息；不能使用测试标签、事后故障说明或由目标窗口生成的解释。
2. 含有 “anomaly”、攻击结论或其他标签性措辞的文本，在进入主实验前必须完成来源与泄漏审计；未通过审计的文本只能保留为数据记录，不能作为模型输入。
3. 模型必须提供 `detect_multi_fit` / `detect_multi_score`，并在前向计算中真实消费文本表征；仅接收参数或在数据加载器中传递占位张量不构成多模态方法。
4. 主结果必须在同一数值骨干、同一切分和同一校准协议下比较 `text_off` 与 `text_on`，从而分离文本条件的实际贡献。

当前的文本 CSV 和所有恢复的 `PatternAD*.sh` 都保留 `--text-name-list`。在完成上述审计与真实融合前，不能把任何结果表述为多模态增益。

## 当前模型

当前唯一活跃模型为 `PatternAD`，代码位于：

```text
PatternAD-main/ts_benchmark/baselines/PatternAD/
```

当前实现先建立时序机制核心：共享时序 Transformer 从可见历史显式提取局部水平、波动、趋势和高频偏离，并在每个时间点和时间尺度上推断变量间的动态有向关系状态。对被掩码的目标值，时间分支和关系分支共同给出条件高斯分布；最终分数为条件负对数似然，而不是手工拼接的残差或尾部统计。

该核心尚未重新实现真实的文本条件分支：现有 `PatternAD` 只有 `detect_fit` / `detect_score`，不能正确执行带 `--text-name-list` 的 benchmark 路径。这是当前待完成的模型工作，不是删除文本输入或将模型定义为单模态的理由。

```text
可见数值模式上下文
    -> 共享时序表征
    -> 多尺度动态有向关系图
    -> 文本/外生语义条件分支（待实现并审计）
    -> 目标盲条件解码
    -> 联合条件似然异常分数
```

模型的三个预声明结构消融为：

```text
full          完整的多尺度动态关系图。
single_scale  移除多尺度关系融合。
no_graph      移除全部跨变量消息，保留相同的时间骨干与概率头。
```

目标值在训练和推理时都被掩码，关系分支不能读取被解释的终点值。这样，跨变量机制不是评分后的修正项，而是表征、解码和训练目标的一部分。

## 评估原则

1. 训练、模型选择和阈值不能使用测试标签；文本同样受这一约束。
2. 先在固定开发集运行 `full`、`single_scale`、`no_graph`，再冻结结构和超参数。
3. 冻结后与重构、图、Transformer 和概率 TSAD 基线在相同数据划分下比较。
4. 报告 AUC-PR、AUC-ROC、VUS-PR、VUS-ROC、运行时间、参数量，以及多 seed 均值和标准差。
5. 真实数据的 GPU 实验尚未运行；目前仅完成时序核心的 CPU 接口与目标盲契约测试。多模态接口和文本泄漏审计完成前，不运行带文本输入的正式对比实验。

既有多模态入口（保持原目录和文件名）：

```bash
cd PatternAD-main
bash scripts/multivariate_detection/detect_label/GECCO_script/PatternAD.sh
```

该入口保留 `--text-name-list`。在 `PatternAD` 的多模态接口重新实现前，它不应被作为可运行的正式实验命令。
